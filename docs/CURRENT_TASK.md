# CURRENT TASK — Stage 1

## 任务名称

四季、草地生物量与春季复苏的生态底层重构。

## 当前状态

代码已经实现，等待人工试玩和诊断报告。没有进入文明、宗教、物种编辑或复杂地形阶段。

## 本阶段唯一目标

解决 Stage 0 的单向消耗问题，使世界具备最基础的季节循环和生态恢复能力：

1. 一年由春、夏、秋、冬组成；
2. 草地不再是“一口删除”的植物个体，而是具有鲜草、枯草、种子库和肥力的斑块；
3. 冬季鲜草转为枯草，地下种子继续保留；
4. 春季从种子库重新萌发；
5. 食草兽只消耗草地生物量；
6. 动物死亡后留下尸体并回归附近土壤；
7. 记录出生、被捕食、饿死、老死和萌发生物量；
8. 创世命题改为让三层生态连续经历3个完整年份。

## 本阶段允许的基础平衡修正

- 动物繁殖受季节、能量、局部食物和种群密度限制；
- 捕猎不再接触即必杀，加入成功率和冷却；
- 猎食兽繁殖需要足够的猎物比例；
- 冬季动物能量消耗略高，但不会直接删除植物种子；
- 生态状态结合资源储备和近期趋势，不再将全灭称为稳定。

这些修正只用于让 Stage 1 能被正常试玩，不代表 Stage 2 的捕食与种群平衡已经完成。

## 禁止事项

本阶段禁止加入：

- 可编辑基因和物种特征；
- 随机突变和进化树；
- 文明、城市、宗教、科技和战争；
- 神力点数、技能树和任务列表；
- 复杂地形、地图编辑和多生态区；
- 折线图、存档、世界种子和服务器功能。

## 人工试玩清单

1. 冬季鲜草减少时，枯草和种子库是否仍然存在；
2. 下一年春季是否出现明显的自然萌发；
3. 鲜草暂时归零时，界面是否显示休眠而非植物彻底灭绝；
4. 食草兽是否仍会短时间无控制爆发；
5. 猎食兽是否仍会接触猎物就必杀；
6. 默认世界能否至少跨越3个完整年份；
7. 世界纪事是否能解释季节变化与年度出生死亡；
8. 本年生命账本是否与画面变化大致一致；
9. 12×速度下是否出现卡死、无限增长或突然全部归零；
10. 导出的诊断报告中是否包含鲜草、枯草、种子、出生和死亡原因。

## 停止条件

完成代码与文档后立即停止，等待玩家提供：

- 新版本截图；
- 至少运行3—6个游戏年的诊断报告；
- 对春季复苏、冬季储备和捕食平衡的直观评价。
