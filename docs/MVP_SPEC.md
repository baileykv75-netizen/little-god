# Little God — Genesis MVP Specification v0.4.1

## 当前目标

制作可直接在浏览器运行的演化生态原型。当前检查点验证：连续、可计算的植物地表能否成为动物觅食、季节变化与生态反馈的共同空间基础。

## 核心规则

- 动物只能依赖局部感知范围内的信息；
- 感知具有距离和判断误差；
- 个体基因型与表现型分离，后代由双亲等位基因重组产生；
- 战力是玩家观察指标，不是动物读取的全局结论；
- 模拟逻辑、规则配置、渲染和界面交互保持边界；
- 植物资源、根系、种子和觅食必须有可解释的收支；
- 不用隐藏数量阈值或固定数字伪造生态稳定。

## 当前世界

- 逻辑尺寸：2048×1280；视口：960×600；
- 初始包含四个局部栖息区；
- 1×下每季约 20 秒，支持 1×、4×、12×；
- 动物总数和各类资源均需有上限与边界检查；
- 根目录可直接作为静态网站部署，不引入框架或构建步骤。

## 当前检查点：连续生态地表

- 64×40 网格，单元 32×32；
- 单元保存鲜草、枯草、根系、种子、肥力、湿度、啃食压力和扰动时间；
- 植被连续平滑渲染，不能显示棋盘格或独立绿色圆圈；
- 根系负责原地复苏，种子负责裸地恢复和扩散；
- 食草兽在当前位置消耗对应地表单元；
- 诊断输出覆盖率、资源预算和啃食热点；
- 详情与唯一任务以 `docs/CURRENT_TASK.md` 为准。

## 当前不做

超凡技能、复杂血脉、群体组织、Observation 和记忆、求偶重构、物种分化、文明、宗教、科技、联机和服务器。

## 完成标准

1. 页面无致命脚本错误；
2. 连续地表、局部觅食、根系、种子和季节变化可在画面与诊断中观察；
3. 相机、暂停、速度、投放、重置、选择和跟随保持可用；
4. 文档与代码状态一致，`docs/HANDOFF.md` 记录已完成内容、限制和试玩结论；
5. 不提前进入下一检查点。
