# Little God — MVP Specification

## 1. MVP目标

制作一个可直接在浏览器运行的生态沙盒原型，验证以下核心体验：

> 玩家投放不同物种、改变少量世界法则，并从可理解的捕食、繁殖、季节与资源循环中观察长期涌现结果。

MVP不追求完整内容，只验证“一个能够自行延续、也可能因玩家规则而崩溃的世界是否值得持续观察”。

## 2. 核心幻想

玩家是一名拥有创造能力、但无法预知结果的神。玩家可以投放草地、食草兽和猎食兽，改变植物生长率、动物繁殖倾向与四季规则，暂停和加速时间，并通过统计、年龄结构和世界纪事理解结果。

玩家不能直接控制单个生物，也不能命令某个物种必须获胜。

## 3. 第一条完整创世命题

> 建立一个能够自我延续的三层生态。

完成条件：

- 草地、食草兽和猎食兽连续自主共存8年；
- 食草兽至少产生8只后代；
- 猎食兽至少产生2只后代；
- 至少发生5次成功捕食；
- 草地至少完成2次冬季后的春季恢复；
- 最近2年没有重新投放动物；
- 共存期间食草兽不低于3只、猎食兽不低于1只。

完成记录是历史成就；若之后生态崩溃，界面必须同时显示“历史已完成”和“当前已崩解”。

## 4. 世界构成

### 草地

每个草地斑块保存鲜草、枯草、地下种子和肥力。春季萌发，夏季生长，秋季结籽，冬季鲜草转为枯草。生长消耗肥力，枯草与尸体分解恢复肥力；尸体不能凭空制造种子。健康草地可以在秋季向附近传播种子。

### 食草兽

寻找鲜草和枯草；遇到猎食兽时短时冲刺并消耗耐力；具有幼体、成年和老年阶段；成年个体在季节、食物、能量和配偶条件满足时繁殖；可能被捕食、饿死或老死。

### 猎食兽

具有巡逻、锁定、追逐、扑击、失败休息、进食和食腐状态；追逐速度高于食草兽短跑，但耐力有限；捕猎并非必然成功；成年个体只有在猎物比例、能量、近期进食与配偶条件满足时繁殖。

## 5. 必须显示的信息

- 世界年份与当前季节；
- 鲜草、枯草和种子储备；
- 两类动物数量和幼体/成年/老年结构；
- 本年分物种出生与死亡原因；
- 捕猎尝试、成功次数和成功率；
- 当前生态结构评分与原因；
- 命题各项条件；
- 精简后的关键世界纪事；
- 可导出的本局诊断报告。

## 6. 生态评分原则

评分必须先判断食物网结构：

- 所有生命灭绝：0；
- 仅剩植物：不高于20；
- 植物与食草兽存在但猎食兽灭绝：不高于55；
- 有猎食兽但无食草兽：不高于15；
- 三层均存在后，才根据资源储备、年龄结构、捕猎效率、出生死亡比和种群趋势计算完整评分。

## 7. 性能与安全边界

- 草地斑块上限56；
- 动物总数上限180；
- 尸体上限80；
- 12×速度下仍可暂停和重置；
- 页面隐藏后不得积累巨大时间步；
- 模拟速度变化不能改变生态规则；
- 世界根目录可直接作为静态网站部署。

## 8. 明确不做

本阶段不包含复杂地形、基因编辑、随机突变、文明、宗教、科技、战争、神力技能树、多地图、多存档、账号、排行榜、联机、外部美术和完整《创世之书》。

## 9. Stage 2验收标准

1. 默认世界前2年内能够发生成功捕食；
2. 捕猎成功率长期大致处于35%—70%；
3. 食草兽前3年产生幼体，猎食兽前5年产生后代；
4. 默认世界随机运行20年时，大多数测试仍保留三层生态；
5. 动物全灭时不得显示“具有恢复力”；
6. 猎食兽未捕食或未产生后代时任务不能完成；
7. 玩家能够从年龄结构和分物种死亡原因理解断代；
8. 新版诊断能回答追逐、扑击、捕猎失败、繁殖、死亡和资源是否不足。
