# Little God

一个以“创世实验”为核心的单人网页沙盒游戏。

玩家不是通过升级成为更强的神，而是通过投放物种、设定世界法则、有限干预与长期观察，回答不同的创世命题，并将独特世界记录进《创世之书》。

## 当前阶段

当前仓库处于 **Genesis v0.2，检查点 2：连续生态地表**。

检查点 1 已完成 2048×1280 逻辑世界、相机缩放与拖动、直接选择、个体跟随和相机诊断。当前唯一开发任务是以 64×40 连续生态网格替换旧的圆形草地斑块，让植物生长、季节变化、局部觅食、根系、种子和诊断共享同一份空间数据。

本检查点验证以下问题：

1. 地表是否连续、可计算、可渲染，而不是多个独立圆形草地；
2. 食草兽是否在真实当前位置消耗地表资源，并形成可观察的啃食缺口；
3. 根系和种子是否分别承担原地复苏与裸地恢复/扩散；
4. 春季返绿与秋冬枯黄是否成为连续的空间过程；
5. 相机、选择、跟随、暂停、速度切换和投放是否保持可用。

## 运行方式

项目不需要安装依赖或构建。

- 直接打开 `index.html`；或
- 使用任意静态服务器托管仓库根目录。

推荐本地运行：

```bash
python -m http.server 8000
```

然后打开 `http://localhost:8000`。

## 项目文档

- `docs/MVP_SPEC.md`：当前原型范围、规则和验收标准；
- `docs/CURRENT_TASK.md`：当前阶段唯一任务与自动验收契约；
- `docs/HANDOFF.md`：已完成内容、已知问题和下一步；
- `AGENTS.md`：长期开发约束。

## 技术原则

- 原生 HTML、CSS、JavaScript；
- 无构建步骤，仓库根目录可直接部署；
- 模拟逻辑与渲染逻辑分离；
- 每个阶段只新增一个可验证的核心能力；
- 不在生态循环成立前加入文明、宗教、科技树或复杂美术。
