# Little God — Genesis v0.2 实施计划

## 总目标

建立“连续生态地图、双尺度观察与局部生命逻辑”的长期地基。

## 检查点1：地图与相机（已完成）

已完成：

- 逻辑世界从960×600扩展为2048×1280；
- 默认全景缩放能够看到完整世界；
- 鼠标滚轮围绕光标缩放；
- 按住空格或鼠标中键拖动地图；
- 右下角提供缩小、放大和返回全景按钮；
- 动物图标缩小，宏观视图中以小型标记显示；
- 左键动物直接观察，点击空地投放当前对象；
- 双击动物进入镜头跟随，Esc退出；
- 选中个体由画布光环高亮；
- 默认世界形成四个局部栖息区；
- 诊断快照加入世界尺寸、相机位置、缩放和跟随目标。

本检查点临时将旧移动单位乘以3，使1×速度下的个体位移可观察。该换算将在G1统一身体方程时迁出渲染模块。

## 检查点2：连续生态地表

### 第1轮实现（已提交）

本轮只替换最核心的数据缺口：让植物、季节更新、局部觅食和资源诊断开始共享同一份64×40空间网格。

已完成：

- 固定64×40生态网格，共2560个地表单元；
- 每个单元保存鲜草、枯草、根系、种子、肥力、湿度、啃食压力和扰动时间；
- 原有播种操作改为对连续网格施加圆形渐变播种刷，而不是新增草地实体；
- 初始四个栖息区由多个相邻网格单元组成；
- 季节更新改为统一网格更新，加入邻域支持、根系复苏、种子萌发、秋季结籽、冬季枯萎、枯草腐解和根系维持损耗；
- 固定网格不会因单元暂时裸露而从数组中删除；
- 现有食草兽继续使用原觅食流程，但目标和实际消耗对象已经是网格单元；
- 现有局部繁殖承载力读取同一网格的局部植物资源；
- 直接减少网格鲜草会记录啃食压力和扰动年份；
- 诊断增加网格尺寸、绿地覆盖率、根系覆盖率、裸地覆盖率、啃食热点和植物资源预算；
- 新增Node回归测试 `tests/vegetation-grid.test.cjs`。

测试命令：

```bash
node --check src/genesis/world-v2.js
node tests/vegetation-grid.test.cjs
```

本轮测试覆盖：

- 网格固定为64×40；
- 2560个单元ID唯一；
- 播种不会增删单元；
- 局部植物资源可以被读取；
- 鲜草被消耗后产生啃食压力；
- 冬季鲜草下降、枯草增加；
- 诊断输出网格结构和覆盖率。

### 当前已知限制

- 当前渲染器仍沿用旧的单元椭圆绘制方法；由于2560个相邻单元重叠，宏观上已经形成连续覆盖，但下一轮仍需改成专用地表渲染器，彻底消除单元形状感；
- 现有食草兽仍会在感知范围内遍历候选网格单元，尚未进行空间索引优化；
- 春季恢复仍沿用旧的绝对增量门槛，尚未改成“冬季损失—春季恢复率”；
- 种子目前只在相邻已有网格上通过萌发扩张，远距离传播和动物携带尚未实现；
- 旧 `patch` 命名为了兼容现有模拟暂时保留，数据本质已经是固定网格单元；
- 活动热区、Observation、求偶重构、技能和物种分化没有进入本轮范围。
